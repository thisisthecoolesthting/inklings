## 1. Why this dispatch

Inklings is voice-first for ages 4â€“8 â€” many of whom can't read yet. The current Studio is tap-only. Add Web Speech API STT so a kid can say "the meadowlands" instead of tapping. **Critically**: STT on young children is unreliable (per the four-way AI review baked into the original Inklings handoff). Tap-button must feel first-class, not a downgrade. Fail-open every failure mode â€” never error to the kid.

Browser support reality check (2026): `SpeechRecognition` (or `webkitSpeechRecognition`) ships in Chrome, Edge, Safari (iOS 14.5+), Samsung Internet. Firefox does NOT ship it. Detect, degrade gracefully, never show a permission wall.

## 2. What you're building

Inside `<SparkyChat>`:

- **Mic button**, big and round, sits ABOVE the choice grid (not replacing it). 88px diameter on mobile, 72px on desktop. Coral background, white mic icon. `aria-label="Tap and talk"` (or i18n future-proof equivalent).
- When tapped:
  - First tap **only** requests `navigator.mediaDevices.getUserMedia({ audio: true })` permission (this kicks the OS prompt). Subsequent taps reuse the grant.
  - Starts a `SpeechRecognition` session: `interimResults: false`, `continuous: false`, `lang: navigator.language || "en-US"`, `maxAlternatives: 3`.
  - Visual: button gets a pulsing coral ring (2s ease-in-out infinite). Inner mic icon stays solid.
  - Auto-stops after first final result OR after 6s timeout.
- On result:
  - Run transcript through `sanitizeChildInput()` from `src/lib/safety.ts`.
  - Fuzzy-match against the visible `beat.choices.map(c => c.label)`. Use Levenshtein OR token-set match (`[label, transcript].split` â†’ Jaccard). Pick closest with normalized distance â‰¤ 0.4.
  - If match: **highlight** the chip (coral ring + scale-105) for 800ms, then call `onChoose(matchedChoice)` â€” same as a tap.
  - If no match: shake the mic button gently (translateX Â±4px, 0.4s), do NOT auto-fail. Kid can tap a chip or re-press mic.
- On any error (no permission, no STT support, network fail): mic button shows a tiny strikethrough icon overlay AND the choice grid stays usable. **Never** display an error message string.

### A â€” new hook: `src/components/studio/use-voice-recognition.ts`

```typescript
"use client";

import { useCallback, useEffect, useRef, useState } from "react";

type VoiceState = "idle" | "listening" | "matched" | "no-match" | "unsupported" | "denied";

interface VoiceResult {
  transcript: string;
  confidence: number;
}

export function useVoiceRecognition(opts?: { lang?: string; timeoutMs?: number }) {
  const [state, setState] = useState<VoiceState>("idle");
  const [lastResult, setLastResult] = useState<VoiceResult | null>(null);
  const recogRef = useRef<any>(null);
  const timeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null);

  // Detect support once on mount
  useEffect(() => {
    if (typeof window === "undefined") return;
    const SR = (window as any).SpeechRecognition || (window as any).webkitSpeechRecognition;
    if (!SR) setState("unsupported");
  }, []);

  const start = useCallback(async () => {
    if (state === "listening") return;
    if (typeof window === "undefined") return;
    const SR = (window as any).SpeechRecognition || (window as any).webkitSpeechRecognition;
    if (!SR) { setState("unsupported"); return; }

    // Permission probe â€” kicks OS prompt if not yet granted
    try {
      const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
      stream.getTracks().forEach(t => t.stop());
    } catch {
      setState("denied");
      return;
    }

    const recog = new SR();
    recog.lang = opts?.lang ?? navigator.language ?? "en-US";
    recog.continuous = false;
    recog.interimResults = false;
    recog.maxAlternatives = 3;

    recog.onresult = (e: any) => {
      const r = e.results?.[0]?.[0];
      if (r) setLastResult({ transcript: r.transcript, confidence: r.confidence ?? 0 });
    };
    recog.onerror = () => setState("idle");
    recog.onend = () => {
      setState((prev) => (prev === "listening" ? "idle" : prev));
      if (timeoutRef.current) { clearTimeout(timeoutRef.current); timeoutRef.current = null; }
    };

    recogRef.current = recog;
    setState("listening");
    setLastResult(null);
    recog.start();
    timeoutRef.current = setTimeout(() => {
      try { recog.stop(); } catch {}
    }, opts?.timeoutMs ?? 6000);
  }, [opts?.lang, opts?.timeoutMs, state]);

  const stop = useCallback(() => {
    try { recogRef.current?.stop(); } catch {}
    if (timeoutRef.current) { clearTimeout(timeoutRef.current); timeoutRef.current = null; }
  }, []);

  return { state, setState, lastResult, start, stop, isSupported: state !== "unsupported" };
}
```

Fuzzy-match implementation (inline in the component, ~15 LOC):

```typescript
function tokenSetMatch(a: string, b: string): number {
  const norm = (s: string) => s.toLowerCase().replace(/[^a-z0-9 ]/g, "").split(/\s+/).filter(Boolean);
  const aSet = new Set(norm(a));
  const bSet = new Set(norm(b));
  const intersect = [...aSet].filter(x => bSet.has(x)).length;
  const union = new Set([...aSet, ...bSet]).size;
  return union === 0 ? 0 : intersect / union; // Jaccard similarity, 0..1
}

function findClosest(transcript: string, choices: SparkyChoice[]): SparkyChoice | null {
  let best: { c: SparkyChoice; score: number } | null = null;
  for (const c of choices) {
    const score = tokenSetMatch(transcript, c.label);
    if (!best || score > best.score) best = { c, score };
  }
  return best && best.score >= 0.5 ? best.c : null;
}
```

Threshold 0.5 not 0.4 â€” STT on small kids garbles enough that we want to be lenient.
